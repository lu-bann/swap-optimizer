Expand description

Type describing a [bitvec::vec::BitVec].

Note

This can only be constructed for TypeInfo in the MetaForm with the bit-vec feature enabled, but can be decoded or deserialized into the PortableForm without this feature.

Creates a new TypeDefBitSequence for the supplied bit order and bit store types.

With the bit-vec feature enabled, the expected usage is to provide either bitvec::order::Lsb0 or bitvec::order::Msb0 as the order type, and then something like u8, u8, or u32 as the store type. Without the bit-vec feature enabled, itโs recommended that your types have identical TypeInfo to those.
